What Is a Context Prompt?

Qu'est-ce qu'un prompt de contexte ?

A context prompt (also called a knowledge base or system prompt) is a set of instructions that Lovable remembers across all future prompts in a project. It acts like a permanent set of rules or guidelines.

Un prompt de contexte (également appelé Knowledge Base ou system prompt) est un ensemble d'instructions que Lovable mémorise pour tous les futurs prompts d'un projet. Il agit comme un ensemble permanent de règles et de directives.

Think of it as the project constitution. Every individual prompt you send is like a specific law — but it must always follow the constitution. The context prompt defines the constants: tech stack, coding style, naming conventions, security rules, design tokens, and more.

Considérez-le comme la constitution du projet. Chaque prompt individuel que vous envoyez est comparable à une loi spécifique — mais elle doit toujours respecter la constitution. Le prompt de contexte définit les constantes : stack technique, style de code, conventions de nommage, règles de sécurité, design tokens, et plus encore.

Without a context prompt, Lovable starts every new prompt with a blank slate. It might choose different styles, different patterns, or inconsistent naming across features. With a context prompt, all output stays aligned.

Sans prompt de contexte, Lovable repart de zéro à chaque nouveau prompt. Il peut choisir des styles différents, des patterns différents ou des conventions de nommage incohérentes d'une fonctionnalité à l'autre. Avec un prompt de contexte, toutes les sorties restent cohérentes.

Persistent Knowledge

Connaissances persistantes

Persistent knowledge means the AI always has access to your context — even when you don't repeat it. This is critical for:

Les connaissances persistantes signifient que l'IA a toujours accès à votre contexte — même lorsque vous ne le répétez pas. C'est essentiel pour :

  • Consistency: Every generated component uses the same patterns.
  • Efficiency: You don't repeat the same instructions every time.
  • Quality: The AI makes better decisions when it knows the full picture.
  • La cohérence : Chaque composant généré utilise les mêmes patterns.
  • L'efficacité : Vous ne répétez pas les mêmes instructions à chaque fois.
  • La qualité : L'IA prend de meilleures décisions lorsqu'elle connaît le contexte complet.

📚 Analogy

Imagine hiring a new developer every day and having to re-explain your project from scratch each morning. That's what happens without a context prompt. With one, it's like having a developer who already read the entire codebase documentation.

📚 Analogie

Imaginez embaucher un nouveau développeur chaque jour et devoir lui réexpliquer votre projet depuis le début chaque matin. C'est ce qui se passe sans prompt de contexte. Avec un prompt de contexte, c'est comme avoir un développeur qui a déjà lu l'intégralité de la documentation du codebase.

Rules vs. Suggestions

Règles vs. suggestions

Not everything in a context prompt carries the same weight. It helps to categorise your instructions into rules (must always follow) and suggestions (preferred but flexible).

Tout ce qui figure dans un prompt de contexte n'a pas la même importance. Il est utile de catégoriser vos instructions en règles (à toujours respecter) et en suggestions (préférées mais flexibles).

Type Example When to Use
Rule "Always use TypeScript. Never use 'any' type." Non-negotiable standards
Rule "All API keys must be in environment variables." Security requirements
Suggestion "Prefer card-based layouts when showing lists." Design preferences
Suggestion "Use toast notifications for success messages." UX patterns
Type Exemple Quand l'utiliser
Règle « Toujours utiliser TypeScript. Ne jamais utiliser le type 'any'. » Standards non négociables
Règle « Toutes les clés API doivent être dans des variables d'environnement. » Exigences de sécurité
Suggestion « Privilégier les mises en page en cartes pour afficher des listes. » Préférences de design
Suggestion « Utiliser des notifications toast pour les messages de succès. » Patterns UX

💡 Tip

Use strong language for rules: "Always", "Never", "Must". Use softer language for suggestions: "Prefer", "When possible", "Default to".

💡 Conseil

Utilisez un langage fort pour les règles : « Toujours », « Jamais », « Doit ». Utilisez un langage plus souple pour les suggestions : « Privilégier », « Si possible », « Par défaut ».

How Context Affects Outputs

Comment le contexte influence les résultats

Here's a concrete example of how context changes the AI's behaviour:

Voici un exemple concret de la façon dont le contexte modifie le comportement de l'IA :

Without Context Prompt

Sans prompt de contexte

Prompt Prompt Build a user profile page.

Result: The AI might use JavaScript (not TypeScript), use a random colour scheme, skip error handling, and use inline styles. It works, but it doesn't match your project.

Résultat : l'IA pourrait utiliser JavaScript (et non TypeScript), choisir une palette de couleurs aléatoire, ignorer la gestion des erreurs et utiliser des styles en ligne. Cela fonctionne, mais ne correspond pas à votre projet.

With Context Prompt

Avec prompt de contexte

Context + Prompt Contexte + Prompt Context (always active): - Use TypeScript with strict mode - Use Tailwind CSS for styling - Primary colour: #7c3aed - All components must handle loading and error states - Use Supabase for data fetching - Follow mobile-first responsive design Prompt: Build a user profile page.

Result: The AI generates a TypeScript React component, uses Tailwind with the correct purple accent, includes loading spinners and error toasts, fetches data from Supabase, and builds mobile-first.

Résultat : l'IA génère un composant React en TypeScript, utilise Tailwind avec le bon accent violet, inclut des indicateurs de chargement et des toasts d'erreur, récupère les données depuis Supabase et adopte une approche mobile-first.

Example: Stack Rules

Exemple : règles de stack

Context Prompt – Stack Prompt de contexte – Stack ## Tech Stack Rules - Frontend: React 18 + TypeScript (strict mode) - Styling: Tailwind CSS (no inline styles, no CSS modules) - State Management: React Query for server state, Zustand for client state - Backend: Supabase (PostgreSQL, Auth, Edge Functions, Storage) - Routing: React Router v6 with lazy-loaded routes - Forms: React Hook Form + Zod validation - Always use named exports (no default exports) - File naming: kebab-case (e.g., user-profile.tsx) - Component naming: PascalCase (e.g., UserProfile)

Example: Security Rules

Exemple : règles de sécurité

Context Prompt – Security Prompt de contexte – Sécurité ## Security Rules - NEVER hardcode API keys, secrets, or passwords in source code - All secrets must use environment variables (import.meta.env) - All Supabase queries must use Row Level Security (RLS) - Sanitize all user inputs before database operations - Authentication tokens must be stored in httpOnly cookies or Supabase session (never in localStorage) - All forms must include CSRF protection - API endpoints must validate request origin

Example: Design Rules

Exemple : règles de design

Context Prompt – Design Prompt de contexte – Design ## Design System Rules - Primary colour: #7c3aed (purple) - Secondary colour: #f472b6 (pink) - Background: #f5f6fa (light grey) - Surface: #ffffff (white cards) - Text: #1e1b4b (dark indigo) - Font: Inter (Google Fonts) - Border radius: 12px for cards, 9999px for buttons - Shadows: use Tailwind shadow-md for cards, shadow-lg for modals - Spacing: use Tailwind spacing scale (p-4, m-6, gap-4) - All pages must be responsive (mobile, tablet, desktop) - Maximum content width: 1200px, centred - Use skeleton loaders during data fetching (not spinners)

⚠️ Warning

A context prompt that is too long (over 2000 words) can confuse the AI or cause it to ignore parts. Keep it focused: prioritise rules over suggestions, and remove any instructions that are obvious or default behaviour.

⚠️ Avertissement

Un prompt de contexte trop long (plus de 2000 mots) peut dérouter l'IA ou l'amener à ignorer certaines parties. Restez concis : privilégiez les règles par rapport aux suggestions, et supprimez toute instruction évidente ou correspondant au comportement par défaut.

How Lovable Interprets Prompts

Comment Lovable interprète les prompts

Lovable's AI reads your prompt, breaks it down into tasks, and generates code for each task. It processes prompts sequentially and literally. This means:

L'IA de Lovable lit votre prompt, le décompose en tâches et génère du code pour chacune d'entre elles. Elle traite les prompts séquentiellement et littéralement. Cela signifie :

  • Every sentence can result in a code decision.
  • Ambiguity leads to the AI choosing for you — and it might choose wrong.
  • Order matters: the AI processes from top to bottom.
  • Shorter, focused prompts yield better results than massive monolithic ones.
  • Chaque phrase peut entraîner une décision de code.
  • L'ambiguïté amène l'IA à choisir à votre place — et elle peut se tromper.
  • L'ordre compte : l'IA traite de haut en bas.
  • Des prompts courts et ciblés donnent de meilleurs résultats que des prompts monolithiques.

📚 Key Concept

Think of Lovable's AI as a very fast, very literal junior developer. It will do exactly what you say — but it won't read between the lines. If you leave something out, it won't be built.

📚 Concept clé

Considérez l'IA de Lovable comme un développeur junior très rapide et très littéral. Elle fera exactement ce que vous dites — mais elle ne lira pas entre les lignes. Si vous omettez quelque chose, cela ne sera pas construit.

Prompt Engineering Basics

Les bases du prompt engineering

Prompt engineering is the skill of writing instructions that produce the best possible output from an AI. Here are the fundamental principles:

Le prompt engineering est l'art de rédiger des instructions qui produisent le meilleur résultat possible d'une IA. Voici les principes fondamentaux :

1. Be Specific

1. Soyez précis

Instead of "add a navbar", say:

Au lieu de « add a navbar », dites :

Prompt Prompt Add a fixed horizontal navigation bar at the top of every page. It should contain: - Logo on the left (link to home page) - Navigation links: Home, Products, Cart, Account - A shopping cart icon with item count badge on the right - On mobile: collapse to a hamburger menu

2. One Task Per Prompt

2. Une tâche par prompt

Avoid sending a 500-word prompt that asks for 10 things at once. Break it down into individual prompts, each focused on a single feature or page.

Évitez d'envoyer un prompt de 500 mots demandant 10 choses à la fois. Décomposez-le en prompts individuels, chacun axé sur une seule fonctionnalité ou page.

3. Use Structured Formatting

3. Utilisez un formatage structuré

Use bullet points, numbered lists, and clear headings. The AI parses structured text much better than long paragraphs.

Utilisez des puces, des listes numérotées et des titres clairs. L'IA analyse le texte structuré bien mieux que de longs paragraphes.

4. Include Constraints

4. Incluez des contraintes

Tell the AI what not to do. For example: "Do NOT add pagination. Show all products on one page."

Dites à l'IA ce qu'elle ne doit pas faire. Par exemple : « Do NOT add pagination. Show all products on one page. »

5. Reference Previous Context

5. Référencez le contexte précédent

If you've already built a page, reference it: "On the existing product detail page, add a reviews section below the specs table."

Si vous avez déjà construit une page, référencez-la : « On the existing product detail page, add a reviews section below the specs table. »

Prompt Structure Template

Modèle de structure de prompt

Use this template for every prompt you send to Lovable:

Utilisez ce modèle pour chaque prompt que vous envoyez à Lovable :

Template Modèle ## [Task Title] ### What to build: [Clear description of the feature or page] ### Requirements: - [Requirement 1] - [Requirement 2] - [Requirement 3] ### Layout / UI: - [Position, size, style details] ### Behaviour: - [What happens on click, load, submit, etc.] ### Constraints: - [What NOT to do] - [Performance / accessibility requirements] ### Data: - [What data is needed, where it comes from]

Prompt 1 : Page d'accueil

Prompt Prompt ## Build the Home Page ### What to build: The main landing page for TechMart e-commerce store. ### Requirements: - Hero section with a large banner image, headline, and CTA button - Featured products section showing 8 products in a 4-column grid - Category navigation section with icon cards (Phones, Laptops, Accessories, Audio) - Footer with links, social icons, and copyright ### Layout / UI: - Full-width hero with gradient overlay on the image - Product cards: image, name, price, "Add to Cart" button - Category cards: icon + label, hover effect with scale ### Behaviour: - CTA button scrolls to featured products - Product cards link to product detail page - Category cards link to filtered category page ### Constraints: - Do NOT add a sidebar - Maximum 8 featured products - Mobile responsive: 2-column grid on tablet, 1-column on phone ### Data: - Products from Supabase "products" table - Categories are hardcoded for now

Prompt 2: Product Detail Page

Prompt 2 : Page de détail produit

Prompt Prompt ## Build the Product Detail Page ### What to build: A detail page for a single product, accessible via /product/:id. ### Requirements: - Image gallery (main image + thumbnail strip) - Product name, price, description - Specifications table (key-value pairs) - Quantity selector (1-10) - "Add to Cart" button - Related products section (4 items from same category) ### Layout / UI: - Two-column layout: images left, details right - On mobile: stack vertically (images on top) - Specs in a striped table - Related products in a horizontal scroll on mobile ### Behaviour: - Click thumbnail to change main image - "Add to Cart" adds to local cart state and shows toast - Related products link to their own detail pages ### Data: - Product data from Supabase by ID - Related products: query same category, exclude current

Adding Authentication

Ajout de l'authentification

Prompt Prompt ## Add User Authentication ### What to build: Email + Google OAuth authentication using Supabase Auth. ### Requirements: - Sign Up page: email, password, confirm password, Google button - Sign In page: email, password, Google button - "Forgot Password" link that triggers Supabase reset email - Protected routes: Cart and Account require login - Show user avatar + name in navbar when logged in ### Behaviour: - After sign up, redirect to home with success toast - After sign in, redirect to previous page or home - Unauthenticated users clicking Cart → redirect to Sign In - Sign out button in account dropdown ### Constraints: - Do NOT add phone number auth - Do NOT add Magic Link auth - Passwords must be minimum 8 characters

Adding a Shopping Cart

Ajout d'un panier d'achat

Prompt Prompt ## Build Shopping Cart Feature ### What to build: A client-side shopping cart stored in localStorage. ### Requirements: - Cart icon in navbar with badge showing item count - Cart page showing all added items - Each item shows: image, name, price, quantity, subtotal - Quantity controls: +/- buttons, minimum 1 - Remove item button - Cart total at the bottom - "Proceed to Checkout" button ### Behaviour: - Adding same product increments quantity - Removing last item shows "Your cart is empty" message - Cart persists across page reloads (localStorage) - "Proceed to Checkout" goes to /checkout (requires auth) ### Constraints: - Do NOT integrate payments yet (that's a separate prompt) - Maximum 99 of any single item

💡 Pro Tip

Send prompts in a logical order: layout first, then data, then interactions, then edge cases. This mirrors how a human developer would build.

💡 Conseil pro

Envoyez les prompts dans un ordre logique : d'abord la mise en page, puis les données, puis les interactions, puis les cas limites. Cela reflète la façon dont un développeur humain construirait.

⚠️ Common Mistake

Don't ask Lovable to "make it look good" or "use best practices". These are too vague. Instead, specify exact colours, spacing, fonts, and layout rules.

⚠️ Erreur courante

Ne demandez pas à Lovable de « make it look good » ou « use best practices ». C'est trop vague. Spécifiez plutôt les couleurs exactes, l'espacement, les polices et les règles de mise en page.

Hosting Model

Modèle d'hébergement

Lovable Cloud is the built-in hosting infrastructure that powers your apps. When you build something in Lovable, it's not just a preview — it's a real, deployed web application running on Lovable's cloud infrastructure.

Lovable Cloud est l'infrastructure d'hébergement intégrée qui fait fonctionner vos applications. Lorsque vous créez quelque chose dans Lovable, ce n'est pas qu'un simple aperçu — c'est une véritable application web déployée, exécutée sur l'infrastructure cloud de Lovable.

Here's how it works under the hood:

Voici comment cela fonctionne en coulisses :

  • Static assets (HTML, CSS, JS, images) are served via a global CDN for fast delivery anywhere in the world.
  • Server-side logic (if any) runs through Supabase Edge Functions — serverless functions deployed at the edge.
  • Database & Auth are handled by Supabase PostgreSQL, with automatic connection pooling and Row Level Security.
  • SSL certificates are provisioned automatically. Every deployed app gets HTTPS out of the box.
  • Ressources statiques (HTML, CSS, JS, images) sont servies via un CDN mondial pour une livraison rapide partout dans le monde.
  • Logique côté serveur (le cas échéant) s'exécute via les Supabase Edge Functions — des fonctions serverless déployées en périphérie.
  • Base de données et authentification sont gérées par Supabase PostgreSQL, avec un pooling de connexions automatique et la Row Level Security.
  • Certificats SSL sont provisionnés automatiquement. Chaque application déployée bénéficie du HTTPS par défaut.

📚 What This Means for You

You never need to configure a server, set up NGINX, manage SSL, or worry about scaling. Lovable Cloud handles all of that. You focus on the product; the platform handles the infrastructure.

📚 Ce que cela signifie pour vous

Vous n'avez jamais besoin de configurer un serveur, de mettre en place NGINX, de gérer le SSL ou de vous soucier de la mise à l'échelle. Lovable Cloud s'occupe de tout cela. Vous vous concentrez sur le produit ; la plateforme gère l'infrastructure.

Environments

Environnements

Professional software development uses environments to separate different stages of your app. Lovable Cloud supports this pattern:

Le développement logiciel professionnel utilise des environnements pour séparer les différentes étapes de votre application. Lovable Cloud prend en charge ce modèle :

Environment Purpose Who Uses It
Preview Live preview during development. Updated with every prompt. You (the builder)
Staging A test environment for QA before going live. You + testers
Production The live version that real users access. Everyone (public)
Environnement Objectif Qui l'utilise
Preview Aperçu en direct pendant le développement. Mis à jour à chaque prompt. Vous (le créateur)
Staging Un environnement de test pour l'assurance qualité avant la mise en ligne. Vous + testeurs
Production La version en ligne accessible aux utilisateurs réels. Tout le monde (public)

Why Environments Matter

Pourquoi les environnements sont importants

Without environments, every change you make is immediately visible to users. This is dangerous — a broken prompt could take down your live app. Environments let you test changes safely before pushing them to production.

Sans environnements, chaque modification que vous effectuez est immédiatement visible par les utilisateurs. C'est risqué — un prompt défaillant pourrait rendre votre application en production inaccessible. Les environnements vous permettent de tester les modifications en toute sécurité avant de les pousser en production.

💡 Best Practice

Always test new features in the preview environment first. Only promote to production when you've verified everything works correctly.

💡 Bonne pratique

Testez toujours les nouvelles fonctionnalités dans l'environnement preview en premier. Ne passez en production que lorsque vous avez vérifié que tout fonctionne correctement.

Deployment Flow

Flux de déploiement

Deploying with Lovable Cloud follows a simple, predictable flow:

Le déploiement avec Lovable Cloud suit un flux simple et prévisible :

  1. Build in Lovable: Use prompts and Visual Mode to create or update your app. Every change is automatically reflected in the preview.
  2. Test in Preview: Click through your app in the preview panel. Test all features, forms, navigation, and edge cases.
  3. Deploy to Staging: When satisfied, click "Deploy" and select the staging environment. Share the staging URL with testers.
  4. QA & Feedback: Testers verify functionality. Collect feedback and address any issues.
  5. Promote to Production: Once approved, promote the staging build to production. Your live URL is updated instantly.
  6. Monitor: After deployment, check for errors, performance issues, or user-reported bugs. Roll back if needed.
  1. Construire dans Lovable : Utilisez les prompts et le Visual Mode pour créer ou mettre à jour votre application. Chaque modification est automatiquement reflétée dans l'aperçu.
  2. Tester dans le Preview : Parcourez votre application dans le panneau d'aperçu. Testez toutes les fonctionnalités, formulaires, la navigation et les cas limites.
  3. Déployer en Staging : Une fois satisfait, cliquez sur « Deploy » et sélectionnez l'environnement staging. Partagez l'URL staging avec les testeurs.
  4. QA et retours : Les testeurs vérifient le fonctionnement. Collectez les retours et corrigez les problèmes identifiés.
  5. Promouvoir en Production : Une fois approuvé, promouvez le build staging en production. Votre URL en ligne est mise à jour instantanément.
  6. Surveiller : Après le déploiement, vérifiez les erreurs, les problèmes de performance ou les bugs signalés par les utilisateurs. Effectuez un rollback si nécessaire.

Version Control

Contrôle de version

Version control means keeping a history of every change made to your project. Lovable provides built-in version control so you can:

Le contrôle de version consiste à conserver un historique de chaque modification apportée à votre projet. Lovable fournit un contrôle de version intégré qui vous permet de :

  • See change history: Every prompt you send creates a new version snapshot.
  • Roll back: If a change breaks something, restore a previous working version.
  • Compare versions: See what changed between any two snapshots.
  • Branch (via GitHub): Lovable integrates with GitHub. You can push your code to a repository and use standard Git workflows.
  • Voir l'historique des modifications : Chaque prompt que vous envoyez crée un nouvel instantané de version.
  • Revenir en arrière : Si une modification casse quelque chose, restaurez une version fonctionnelle précédente.
  • Comparer les versions : Visualisez ce qui a changé entre deux instantanés.
  • Créer des branches (via GitHub) : Lovable s'intègre avec GitHub. Vous pouvez pousser votre code vers un dépôt et utiliser les workflows Git standards.

Lovable + GitHub Integration

Intégration Lovable + GitHub

Lovable can sync your project with a GitHub repository. This gives you:

Lovable peut synchroniser votre projet avec un dépôt GitHub. Cela vous offre :

  • Full Git history with commits for each change
  • Ability to clone the repo and work locally
  • Pull requests and code reviews if you have a team
  • CI/CD pipeline integration if needed
  • Un historique Git complet avec des commits pour chaque modification
  • La possibilité de cloner le dépôt et de travailler localement
  • Des pull requests et des revues de code si vous avez une équipe
  • L'intégration de pipelines CI/CD si nécessaire
Example Git Flow Exemple de flux Git Lovable generates code ↓ Auto-commit to GitHub (main branch) ↓ You can create feature branches locally ↓ Merge back to main ↓ Lovable syncs with latest main

⚠️ Important

If you make changes in GitHub directly (outside Lovable), make sure to sync before building more in Lovable to avoid conflicts.

⚠️ Important

Si vous effectuez des modifications directement dans GitHub (en dehors de Lovable), assurez-vous de synchroniser avant de continuer à construire dans Lovable pour éviter les conflits.

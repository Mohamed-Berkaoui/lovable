Visual Mode

Click, drag, and tweak your UI visually — no code required. Understand when and how to use it.

🕑 Estimated reading time: 9 minutes

What Is Visual Mode?

Visual Mode is a feature in Lovable that lets you edit your app's interface by directly clicking on elements in the preview, rather than writing text prompts. It provides a point-and-click editing experience similar to Figma or a WYSIWYG editor.

When you activate Visual Mode, hovering over any element in the preview highlights it. Clicking selects it, and a panel appears with editable properties: text content, colours, spacing, font sizes, borders, and more.

Under the hood, Lovable translates your visual changes into actual code changes. So when you drag a button 20 pixels to the right in Visual Mode, the underlying CSS/JSX is updated accordingly.

When to Use Visual Mode

Visual Mode is ideal for small, visual adjustments. It's not designed for adding new features or changing business logic. Here's a guide:

Use Visual Mode For Use Text Prompts For
Changing button colour Adding a new button with logic
Adjusting padding/margins Restructuring the entire layout
Editing text content Adding dynamic text from a database
Changing font size Implementing a theme switcher
Tweaking border radius Adding form validation
Hiding/showing an element Conditional rendering based on data

💡 Rule of Thumb

If the change is purely cosmetic and takes less than 5 seconds to describe, use Visual Mode. If the change involves logic, data, or multiple elements, use a text prompt.

Visual vs. Textual Control

Both visual and textual editing have strengths. The best approach is to use them together:

Textual Prompts — Best For:

  • Building new features from scratch
  • Implementing complex logic (auth, data fetching, state management)
  • Making changes that affect multiple pages simultaneously
  • Creating reusable components with specific props

Visual Mode — Best For:

  • Fine-tuning spacing, alignment, and sizing
  • Quick colour adjustments
  • Editing static text (headings, labels, placeholder text)
  • Previewing changes instantly before committing

The Ideal Workflow

  1. Build with text prompts: Generate the feature structure and logic.
  2. Review in preview: Check the output visually.
  3. Polish with Visual Mode: Adjust spacing, colours, text as needed.
  4. Refine with text prompts: If bigger changes are needed, go back to text.

Example: Editing Layout with Visual Mode

Scenario

You generated a product card component with a text prompt. The card works correctly, but the image is too small, the padding is uneven, and the "Add to Cart" button should be green instead of blue.

Visual Mode Steps

  1. Activate Visual Mode by clicking the toggle in Lovable's toolbar.
  2. Click the product image. A properties panel opens. Change height from 200px to 280px. Changes apply instantly in the preview.
  3. Click the card container. Adjust padding from 12px to 20px on all sides. The spacing now looks balanced.
  4. Click the "Add to Cart" button. Change the background colour from #3b82f6 (blue) to #16a34a (green). The button updates immediately.
  5. Review the changes. If satisfied, they're automatically saved to code.

📚 Note

Every change you make in Visual Mode is reflected in the actual source code. You can switch to the code view to see exactly what CSS or JSX was modified.

Example: Component Tuning

Scenario

Your navigation bar looks good on desktop but the links are too small on tablet, and the search bar needs more width.

Visual Mode Steps

  1. Switch to tablet preview using Lovable's responsive toggle.
  2. Click a nav link. Increase font size from 14px to 16px. Increase padding from 4px 8px to 8px 16px.
  3. Click the search bar. Change width from 200px to 100% (or set flex-grow: 1) so it fills available space.
  4. Review on mobile view to ensure changes don't break the hamburger menu layout.

⚠️ Warning

Visual Mode changes can sometimes conflict with responsive styles defined via text prompts. After using Visual Mode, always check multiple screen sizes to ensure nothing broke.

Section Summary

  1. Visual Mode lets you edit UI elements by clicking directly in the preview.
  2. Use it for cosmetic tweaks: colours, spacing, text, sizing.
  3. Use text prompts for logic, data, and multi-element changes.
  4. The ideal workflow: build with text → polish with visual → refine with text.
  5. Always check responsive views after visual edits.

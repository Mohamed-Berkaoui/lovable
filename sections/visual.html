Cliquez, glissez et ajustez votre interface visuellement — sans écrire de code. Découvrez quand et comment l'utiliser.

What Is Visual Mode?

Qu'est-ce que le Mode Visuel ?

Visual Mode is a feature in Lovable that lets you edit your app's interface by directly clicking on elements in the preview, rather than writing text prompts. It provides a point-and-click editing experience similar to Figma or a WYSIWYG editor.

Le Mode Visuel est une fonctionnalité de Lovable qui vous permet de modifier l'interface de votre application en cliquant directement sur les éléments dans l'aperçu, plutôt qu'en rédigeant des prompts textuels. Il offre une expérience d'édition pointer-cliquer similaire à Figma ou un éditeur WYSIWYG.

When you activate Visual Mode, hovering over any element in the preview highlights it. Clicking selects it, and a panel appears with editable properties: text content, colours, spacing, font sizes, borders, and more.

Lorsque vous activez le Mode Visuel, le survol de n'importe quel élément dans l'aperçu le met en surbrillance. Un clic le sélectionne, et un panneau apparaît avec les propriétés modifiables : contenu textuel, couleurs, espacement, tailles de police, bordures, etc.

Under the hood, Lovable translates your visual changes into actual code changes. So when you drag a button 20 pixels to the right in Visual Mode, the underlying CSS/JSX is updated accordingly.

En coulisses, Lovable traduit vos modifications visuelles en véritables changements de code. Ainsi, lorsque vous déplacez un bouton de 20 pixels vers la droite en Mode Visuel, le CSS/JSX sous-jacent est mis à jour en conséquence.

When to Use Visual Mode

Quand utiliser le Mode Visuel

Visual Mode is ideal for small, visual adjustments. It's not designed for adding new features or changing business logic. Here's a guide:

Le Mode Visuel est idéal pour les petits ajustements visuels. Il n'est pas conçu pour ajouter de nouvelles fonctionnalités ou modifier la logique métier. Voici un guide :

Use Visual Mode For Use Text Prompts For
Changing button colour Adding a new button with logic
Adjusting padding/margins Restructuring the entire layout
Editing text content Adding dynamic text from a database
Changing font size Implementing a theme switcher
Tweaking border radius Adding form validation
Hiding/showing an element Conditional rendering based on data
Utiliser le Mode Visuel pour Utiliser les prompts textuels pour
Changer la couleur d'un bouton Ajouter un nouveau bouton avec de la logique
Ajuster le padding/les marges Restructurer l'ensemble de la mise en page
Modifier le contenu textuel Ajouter du texte dynamique depuis une base de données
Changer la taille de police Implémenter un sélecteur de thème
Ajuster le border radius Ajouter la validation de formulaire
Masquer/afficher un élément Rendu conditionnel basé sur les données

💡 Rule of Thumb

If the change is purely cosmetic and takes less than 5 seconds to describe, use Visual Mode. If the change involves logic, data, or multiple elements, use a text prompt.

💡 Règle générale

Si la modification est purement esthétique et prend moins de 5 secondes à décrire, utilisez le Mode Visuel. Si elle implique de la logique, des données ou plusieurs éléments, utilisez un prompt textuel.

Visual vs. Textual Control

Contrôle visuel vs. textuel

Both visual and textual editing have strengths. The best approach is to use them together:

L'édition visuelle et textuelle ont chacune leurs forces. La meilleure approche est de les utiliser ensemble :

Textual Prompts — Best For:

Prompts textuels — Idéaux pour :

  • Building new features from scratch
  • Implementing complex logic (auth, data fetching, state management)
  • Making changes that affect multiple pages simultaneously
  • Creating reusable components with specific props
  • Construire de nouvelles fonctionnalités à partir de zéro
  • Implémenter une logique complexe (auth, data fetching, state management)
  • Effectuer des modifications affectant plusieurs pages simultanément
  • Créer des composants réutilisables avec des props spécifiques

Visual Mode — Best For:

Mode Visuel — Idéal pour :

  • Fine-tuning spacing, alignment, and sizing
  • Quick colour adjustments
  • Editing static text (headings, labels, placeholder text)
  • Previewing changes instantly before committing
  • Peaufiner l'espacement, l'alignement et le dimensionnement
  • Ajustements rapides des couleurs
  • Modifier le texte statique (titres, libellés, textes de remplacement)
  • Prévisualiser les modifications instantanément avant de valider

The Ideal Workflow

Le flux de travail idéal

  1. Build with text prompts: Generate the feature structure and logic.
  2. Review in preview: Check the output visually.
  3. Polish with Visual Mode: Adjust spacing, colours, text as needed.
  4. Refine with text prompts: If bigger changes are needed, go back to text.
  1. Construire avec les prompts textuels : Générer la structure et la logique de la fonctionnalité.
  2. Vérifier dans l'aperçu : Contrôler le résultat visuellement.
  3. Peaufiner avec le Mode Visuel : Ajuster l'espacement, les couleurs et le texte selon les besoins.
  4. Affiner avec les prompts textuels : Si des modifications plus importantes sont nécessaires, revenir au texte.

Example: Editing Layout with Visual Mode

Exemple : Modifier la mise en page avec le Mode Visuel

Scenario

Scénario

You generated a product card component with a text prompt. The card works correctly, but the image is too small, the padding is uneven, and the "Add to Cart" button should be green instead of blue.

Vous avez généré un composant de carte produit avec un prompt textuel. La carte fonctionne correctement, mais l'image est trop petite, le padding est inégal, et le bouton « Add to Cart » devrait être vert au lieu de bleu.

Visual Mode Steps

Étapes en Mode Visuel

  1. Activate Visual Mode by clicking the toggle in Lovable's toolbar.
  2. Click the product image. A properties panel opens. Change height from 200px to 280px. Changes apply instantly in the preview.
  3. Click the card container. Adjust padding from 12px to 20px on all sides. The spacing now looks balanced.
  4. Click the "Add to Cart" button. Change the background colour from #3b82f6 (blue) to #16a34a (green). The button updates immediately.
  5. Review the changes. If satisfied, they're automatically saved to code.
  1. Activer le Mode Visuel en cliquant sur le bouton bascule dans la barre d'outils de Lovable.
  2. Cliquer sur l'image du produit. Un panneau de propriétés s'ouvre. Modifier la hauteur de 200px à 280px. Les changements s'appliquent instantanément dans l'aperçu.
  3. Cliquer sur le conteneur de la carte. Ajuster le padding de 12px à 20px sur tous les côtés. L'espacement est maintenant équilibré.
  4. Cliquer sur le bouton « Add to Cart ». Changer la couleur de fond de #3b82f6 (bleu) à #16a34a (vert). Le bouton se met à jour immédiatement.
  5. Vérifier les modifications. Si vous êtes satisfait, elles sont automatiquement enregistrées dans le code.

📚 Note

Every change you make in Visual Mode is reflected in the actual source code. You can switch to the code view to see exactly what CSS or JSX was modified.

📚 Remarque

Chaque modification effectuée en Mode Visuel est reflétée dans le code source réel. Vous pouvez basculer vers la vue code pour voir exactement quel CSS ou JSX a été modifié.

Example: Component Tuning

Exemple : Ajustement de composants

Scenario

Scénario

Your navigation bar looks good on desktop but the links are too small on tablet, and the search bar needs more width.

Votre barre de navigation est bien sur desktop, mais les liens sont trop petits sur tablette, et la barre de recherche a besoin de plus de largeur.

Visual Mode Steps

Étapes en Mode Visuel

  1. Switch to tablet preview using Lovable's responsive toggle.
  2. Click a nav link. Increase font size from 14px to 16px. Increase padding from 4px 8px to 8px 16px.
  3. Click the search bar. Change width from 200px to 100% (or set flex-grow: 1) so it fills available space.
  4. Review on mobile view to ensure changes don't break the hamburger menu layout.
  1. Passer à l'aperçu tablette en utilisant le sélecteur responsive de Lovable.
  2. Cliquer sur un lien de navigation. Augmenter la taille de police de 14px à 16px. Augmenter le padding de 4px 8px à 8px 16px.
  3. Cliquer sur la barre de recherche. Changer la largeur de 200px à 100 % (ou définir flex-grow: 1) pour qu'elle occupe l'espace disponible.
  4. Vérifier en vue mobile pour s'assurer que les modifications ne cassent pas la mise en page du menu hamburger.

⚠️ Warning

Visual Mode changes can sometimes conflict with responsive styles defined via text prompts. After using Visual Mode, always check multiple screen sizes to ensure nothing broke.

⚠️ Avertissement

Les modifications du Mode Visuel peuvent parfois entrer en conflit avec les styles responsive définis via les prompts textuels. Après avoir utilisé le Mode Visuel, vérifiez toujours plusieurs tailles d'écran pour vous assurer que rien n'est cassé.

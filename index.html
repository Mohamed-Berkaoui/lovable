Section 02

Project Preparation

Préparation du projet

Great apps begin with great preparation define your problem, users, and goals before building.

Les grandes applications commencent par une bonne préparation — définissez votre problème, vos utilisateurs et vos objectifs avant de construire.

  • Write a 2-sentence problem statement before opening Lovable
  • Create 23 realistic user personas to ground every decision
  • Define measurable success metrics (acquisition, activation, retention)
  • Identify MVP features vs. nice-to-haves ruthlessly cut scope
  • Avoid feature overload, vague goals, and skipped research
  • Rédigez un énoncé du problème en 2 phrases avant d'ouvrir Lovable
  • Créez 2-3 personas utilisateurs réalistes pour ancrer chaque décision
  • Définissez des métriques de succès mesurables (acquisition, activation, rétention)
  • Identifiez les fonctionnalités MVP vs. les « nice-to-have » — réduisez sans pitié
  • Évitez la surcharge de fonctionnalités, les objectifs vagues et la recherche bâclée

Section 03

PRD Creation

Création du PRD

The Product Requirements Document is your blueprint. Without it, you're building blind.

Le document de spécifications produit est votre plan. Sans lui, vous construisez à l'aveugle.

  • Project overview, problem statement, and target users
  • Feature list scoped to MVP only no extras
  • User flows, navigation, and page-by-page screen elements
  • Tech stack, database schema, and design tokens
  • Success metrics and explicit out-of-scope list
  • Vue d'ensemble du projet, énoncé du problème et utilisateurs cibles
  • Liste de fonctionnalités limitée au MVP uniquement — pas d'extras
  • Parcours utilisateurs, navigation et éléments d'écran page par page
  • Stack technique, schéma de base de données et design tokens
  • Métriques de succès et liste explicite des éléments hors périmètre

Section 04

PRD → Lovable Prompts

PRD → Prompts Lovable

Turn your PRD into precise, structured prompts that Lovable executes with accuracy.

Transformez votre PRD en prompts précis et structurés que Lovable exécute avec exactitude.

  • One feature per prompt never bundle multiple requirements
  • Use bullet-point lists, not paragraphs, inside each prompt
  • State explicitly what NOT to change or break
  • Reference existing components and pages by name
  • Build in logical order: layout → data → logic → polish
  • Une fonctionnalité par prompt — ne jamais regrouper plusieurs exigences
  • Utilisez des listes à puces, pas des paragraphes, dans chaque prompt
  • Indiquez explicitement ce qu'il ne faut PAS modifier ou casser
  • Référencez les composants et pages existants par leur nom
  • Construisez dans l'ordre logique : mise en page → données → logique → finition

Section 05

Context Prompt (Knowledge Base)

Prompt de contexte (Base de connaissances)

Permanent project-wide instructions that keep every AI output consistent and on-brand.

Des instructions permanentes à l'échelle du projet pour garder chaque sortie IA cohérente et fidèle à la marque.

  • Lock in your tech stack: React, TypeScript, Tailwind, Supabase
  • Define security rules never expose API keys, always use RLS
  • Set design tokens: colours, fonts, spacing, and responsive behaviour
  • Document naming conventions and code style preferences
  • Keep it under 2,000 characters for optimal AI comprehension
  • Verrouillez votre stack technique : React, TypeScript, Tailwind, Supabase
  • Définissez les règles de sécurité — ne jamais exposer les clés API, toujours utiliser RLS
  • Fixez les design tokens : couleurs, polices, espacement et comportement responsive
  • Documentez les conventions de nommage et les préférences de style de code
  • Gardez-le sous 2 000 caractères pour une compréhension IA optimale

Section 06

Iteration & Refinement

Itération & Perfectionnement

Nothing is right on the first try. Learn to use the feedback loop to reach perfection, fast.

Rien n'est parfait du premier coup. Apprenez à utiliser la boucle de feedback pour atteindre la perfection, rapidement.

  • Follow the Prompt → Review → Identify → Refine → Repeat cycle
  • Use the "Keep / Change / Remove" format for clear feedback
  • Fix one bug per prompt surgical, not sweeping
  • Test on mobile after every single change
  • Define a "good enough" threshold to avoid endless polish
  • Suivez le cycle Prompt → Révision → Identification → Affinage → Répétition
  • Utilisez le format « Garder / Modifier / Supprimer » pour un feedback clair
  • Corrigez un seul bug par prompt — chirurgical, pas massif
  • Testez sur mobile après chaque modification
  • Définissez un seuil « assez bien » pour éviter le polissage sans fin

Section 07

Visual Mode

Mode Visuel

Click directly on any element in the preview to adjust colours, spacing, text, and more.

Cliquez directement sur n'importe quel élément dans l'aperçu pour ajuster les couleurs, l'espacement, le texte, et plus encore.

  • Use Visual Mode for colours, padding, font sizes, and static text
  • Use text prompts for logic, new components, and multi-page changes
  • Combine both: rough prompt to build, Visual Mode to fine-tune
  • Changes in Visual Mode are tracked in version history
  • Great for fast design tweaks without re-prompting from scratch
  • Utilisez le Mode Visuel pour les couleurs, marges, tailles de police et texte statique
  • Utilisez les prompts texte pour la logique, nouveaux composants et changements multi-pages
  • Combinez les deux : prompt brut pour construire, Mode Visuel pour affiner
  • Les modifications en Mode Visuel sont suivies dans l'historique des versions
  • Idéal pour des ajustements de design rapides sans re-prompter de zéro

Section 10

API Integration

Intégration API

Connect your app to the world: external APIs, AI services, payment systems, and more.

Connectez votre application au monde : APIs externes, services IA, systèmes de paiement, et plus.

  • Build a Supabase Edge Function as a secure server-side proxy
  • Store all API keys in environment secrets never in frontend code
  • Call your Edge Function from the React frontend
  • Handle success, error, and loading states for every API call
  • Connect to OpenAI, Stripe, Resend, and more with the same pattern
  • Créez une Edge Function Supabase comme proxy sécurisé côté serveur
  • Stockez toutes les clés API dans les secrets d'environnement — jamais dans le code frontend
  • Appelez votre Edge Function depuis le frontend React
  • Gérez les états succès, erreur et chargement pour chaque appel API
  • Connectez-vous à OpenAI, Stripe, Resend et plus avec le même pattern

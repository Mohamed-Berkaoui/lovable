Section 02

Project Preparation

Great apps begin with great preparation define your problem, users, and goals before building.

  • Write a 2-sentence problem statement before opening Lovable
  • Create 23 realistic user personas to ground every decision
  • Define measurable success metrics (acquisition, activation, retention)
  • Identify MVP features vs. nice-to-haves ruthlessly cut scope
  • Avoid feature overload, vague goals, and skipped research
Deep Dive →
Section 03

PRD Creation

The Product Requirements Document is your blueprint. Without it, you're building blind.

  • Project overview, problem statement, and target users
  • Feature list scoped to MVP only no extras
  • User flows, navigation, and page-by-page screen elements
  • Tech stack, database schema, and design tokens
  • Success metrics and explicit out-of-scope list
Deep Dive →
Section 04

PRD → Lovable Prompts

Turn your PRD into precise, structured prompts that Lovable executes with accuracy.

  • One feature per prompt never bundle multiple requirements
  • Use bullet-point lists, not paragraphs, inside each prompt
  • State explicitly what NOT to change or break
  • Reference existing components and pages by name
  • Build in logical order: layout → data → logic → polish
Deep Dive →
Section 05

Context Prompt (Knowledge Base)

Permanent project-wide instructions that keep every AI output consistent and on-brand.

  • Lock in your tech stack: React, TypeScript, Tailwind, Supabase
  • Define security rules never expose API keys, always use RLS
  • Set design tokens: colours, fonts, spacing, and responsive behaviour
  • Document naming conventions and code style preferences
  • Keep it under 2,000 characters for optimal AI comprehension
Deep Dive →
Section 06

Iteration & Refinement

Nothing is right on the first try. Learn to use the feedback loop to reach perfection, fast.

  • Follow the Prompt → Review → Identify → Refine → Repeat cycle
  • Use the "Keep / Change / Remove" format for clear feedback
  • Fix one bug per prompt surgical, not sweeping
  • Test on mobile after every single change
  • Define a "good enough" threshold to avoid endless polish
Deep Dive →
Section 07

Visual Mode

Click directly on any element in the preview to adjust colours, spacing, text, and more.

  • Use Visual Mode for colours, padding, font sizes, and static text
  • Use text prompts for logic, new components, and multi-page changes
  • Combine both: rough prompt to build, Visual Mode to fine-tune
  • Changes in Visual Mode are tracked in version history
  • Great for fast design tweaks without re-prompting from scratch

API Integration

Connect your app to the world: external APIs, AI services, payment systems, and more.
